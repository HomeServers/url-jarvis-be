Link Mind 개인정보 처리방침

시행일: 2026년 2월 24일

Link Mind(이하 "본 서비스")는 사용자의 개인정보를 중요하게 생각하며, 「개인정보 보호법」에 따라 아래와 같이 개인정보 처리방침을 안내합니다.

1. 수집하는 개인정보

항목 수집 목적 수집 방법
Google 계정 이메일 사용자 인증 및 계정 식별 Google OAuth 로그인 시
Google 프로필 정보 (이름) 서비스 내 사용자 표시 Google OAuth 로그인 시
저장한 웹페이지 URL URL 저장 및 분석 서비스 제공 사용자가 직접 저장 요청 시

2. 개인정보의 이용 목적

  • 사용자 인증 및 로그인 세션 관리
  • 사용자가 요청한 URL의 저장 및 관리
  • 서비스 운영 및 개선

3. 개인정보의 보관 및 저장

3.1 로컬 저장 (Chrome 브라우저)

  • 인증 토큰 (Access Token, Refresh Token): Chrome 로컬 스토리지에 저장되며, 로그아웃 시 즉시 삭제됩니다.
  • 서버 URL 설정: Chrome 동기화 스토리지에 저장됩니다.

3.2 서버 저장

  • 사용자가 저장한 URL 데이터는 사용자가 설정한 Link Mind 서버에 전송 및 저장됩니다.
  • 서버의 데이터 보관 정책은 해당 서버 운영자의 정책을 따릅니다.

4. 제3자 제공

본 서비스는 사용자의 개인정보를 제3자에게 제공하지 않습니다. 다만, 다음의 경우는 예외로 합니다:

  • Google OAuth 인증을 위해 Google에 인증 요청을 전송합니다.
  • 법령에 의해 요구되는 경우

5. 외부 서비스

서비스 목적 개인정보 처리방침
Google OAuth 2.0 사용자 인증 Google 개인정보 처리방침

6. 사용자의 권리

  • 로그아웃: 확장 프로그램 팝업 또는 설정 페이지에서 로그아웃하면 로컬에 저장된 모든 인증 정보가 삭제됩니다.
  • 확장 프로그램 삭제: Chrome에서 확장 프로그램을 제거하면 로컬에 저장된 모든 데이터가 삭제됩니다.
  • 계정 삭제 요청: 서버에 저장된 데이터의 삭제를 원하시면 아래 연락처로 요청해 주세요.

7. 브라우저 권한 안내

권한 사용 목적
activeTab 현재 탭의 URL을 읽어 서버에 저장하기 위해 사용됩니다.
storage 인증 토큰과 서버 설정을 브라우저에 저장하기 위해 사용됩니다.
identity Google OAuth 로그인 흐름을 처리하기 위해 사용됩니다.
host_permissions (all URLs) 사용자가 설정한 서버로 데이터를 전송하기 위해 필요합니다.

8. 개인정보 처리방침의 변경

본 처리방침이 변경되는 경우, 변경 사항을 이 페이지에 게시하며 시행일을 업데이트합니다.

Link Mind Privacy Policy

Effective Date: February 24, 2026

Link Mind ("the Service") values your privacy. This Privacy Policy describes how we collect, use, and protect your personal information when you use the Link Mind Chrome Extension.

1. Information We Collect

Data Purpose Collection Method
Google account email User authentication and identification Google OAuth login
Google profile information (name) Display within the service Google OAuth login
Saved webpage URLs URL saving and analysis service User-initiated save action

2. How We Use Your Information

  • User authentication and login session management
  • Saving and managing URLs as requested by the user
  • Service operation and improvement

3. Data Storage

3.1 Local Storage (Chrome Browser)

  • Authentication tokens (Access Token, Refresh Token): Stored in Chrome local storage and deleted immediately upon logout.
  • Server URL setting: Stored in Chrome sync storage.

3.2 Server Storage

  • URLs saved by the user are transmitted to and stored on the Link Mind server configured by the user.
  • Server data retention policies are governed by the respective server operator.

4. Third-Party Sharing

We do not share your personal information with third parties, except in the following cases:

  • Authentication requests are sent to Google for OAuth login.
  • When required by law

5. External Services

Service Purpose Privacy Policy
Google OAuth 2.0 User authentication Google Privacy Policy

6. Your Rights

  • Logout: Logging out from the extension popup or settings page deletes all locally stored authentication data.
  • Uninstall: Removing the extension from Chrome deletes all locally stored data.
  • Account deletion: To request deletion of server-stored data, please contact us using the information below.

7. Browser Permissions

Permission Purpose
activeTab Used to read the current tab's URL for saving to the server.
storage Used to store authentication tokens and server settings in the browser.
identity Used to handle the Google OAuth login flow.
host_permissions (all URLs) Required to send data to the user-configured server.

8. Changes to This Policy

If this Privacy Policy is updated, the changes will be posted on this page with an updated effective date.
